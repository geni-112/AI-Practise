# Scenario Demo Quality Checklist

Use this checklist before final response.

## Purpose

- The first viewport says what the demo is for.
- The target user and business task are visible or obvious.
- The page explains input, processing, and output in plain language.
- Important acronyms are expanded or avoided.

## Workflow

- There is at least one meaningful action button or control.
- Sample data is realistic for the domain.
- Output changes or renders based on user selection.
- Empty, warning, review, or quality states are represented when relevant.

## Architecture

- Source systems, orchestration, service/AI layer, business apps, and feedback loop are visible when relevant.
- Diagram labels are concise and stakeholder-readable.
- Arrows do not obscure labels or run through unrelated boxes.

## Visual QA

- No overlapping UI elements at desktop width.
- Mobile layout stacks cleanly.
- Tables or dense grids scroll within their own container when needed.
- Body has no unexpected horizontal overflow.
- Console has no errors or warnings from the demo code.

## Handoff

- Provide the preview URL or local file path.
- List the important changed files.
- Mention verification performed and any limitation, such as screenshot capture failure.
